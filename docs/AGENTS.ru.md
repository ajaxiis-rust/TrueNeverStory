# Справка по агентам (v0.26.0)

TrueNeverStory использует мультиагентную архитектуру, где каждый агент отвечает за определённый аспект нарратива. Начиная с v0.26.0, движок использует **6 основных агентов** (The Big Six) плюс **5 специализированных агентов** и **систему диалогов**.

## Глобальные переменные

Эти переменные доступны большинству агентов через контекст состояния мира:

| Переменная | Описание |
|-----------|----------|
| `{world_name}` | Имя текущего мира (из world_frame.json) |
| `{time}` | Текущее время истории (ISO строка) |
| `{location}` | Текущая локация персонажа |
| `{character}` | Имя активного персонажа |
| `{role}` | Роль игрока (протагонист, наблюдатель и т.д.) |
| `{rules}` | Правила мира (законы магии, социальные нормы и т.д.) |
| `{timeline}` | Недавние события мира (последние 5 от летописца) |
| `{memories}` | Недавние воспоминания ролевой игры |
| `{facts}` | Установленные факты мира |
| `{npcs}` | Имена ближайших NPC |
| `{history}` | Последняя история разговора (последние 3 обмена) |
| `{events}` | Недавние события (контекстно-зависимые, последние 3-5) |
| `{world_state}` | Сводка текущего состояния мира |
| `{world_context}` | Контекст мира для исследований |

## Агенты

### Рассказчик (`narrator`)

**Описание:** Главный рассказчик. Генерирует повествование мира из контекста истории.

**Переменные шаблона:**
`{world_name}` `{time}` `{location}` `{character}` `{role}` `{rules}` `{timeline}` `{memories}` `{facts}` `{npcs}` `{history}`

**Системный промпт:** Определяет рассказчика как искусного storyteller. Пишет яркую, атмосферную прозу от второго/третьего лица. Никогда не выходит из роли.

**Температура:** 0.8 | **Макс. токенов:** 4096 | **Приоритет:** 10 (наивысший)

---

### Режиссёр (`director`)

**Описание:** Инъекция сюжетных битов. Интегрирует драматические моменты в нарратив.

**Переменные шаблона:**
`{narrative}` `{beat}`

| Переменная | Описание |
|-----------|----------|
| `{narrative}` | Текущий текст нарратива для инъекции бита |
| `{beat}` | Описание сюжетного бита (инцидент, откровение, неудача и т.д.) |

**Температура:** 0.7 | **Макс. токенов:** 2048 | **Приоритет:** 8

---

### Генератор сцен (`scene`)

**Описание:** Описание переходов между сценами при перемещении персонажей.

**Переменные шаблона:**
`{character}` `{origin}` `{destination}` `{rules}` `{events}`

| Переменная | Описание |
|-----------|----------|
| `{origin}` | Текущая локация (откуда персонаж уходит) |
| `{destination}` | Целевая локация (куда персонаж идёт) |

**Температура:** 0.8 | **Макс. токенов:** 2048 | **Приоритет:** 7

---

### Агент NPC (`npc`)

**Описание:** Диалоги и реакции NPC. Играет отдельных персонажей.

**Переменные шаблона:**
`{npc_name}` `{npc_personality}` `{player}` `{location}` `{relationship}` `{events}` `{line}`

| Переменная | Описание |
|-----------|----------|
| `{npc_name}` | Имя NPC, за которого играет агент |
| `{npc_personality}` | Черты личности NPC (из профиля сущности) |
| `{player}` | Имя персонажа игрока |
| `{relationship}` | Отношение к игроку (друг, нейтрал, враг и т.д.) |
| `{line}` | Что игрок сказал NPC |

**Температура:** 0.7 | **Макс. токенов:** 1024 | **Приоритет:** 9

---

### Летописец (`chronicler`)

**Описание:** Управление таймлайном. Резюмирует события и ведёт хронику мира.

**Переменные шаблона:**
`{events}` `{timeline}`

| Переменная | Описание |
|-----------|----------|
| `{events}` | Новые события для летописи (действия, перемещения, диалоги) |
| `{timeline}` | Существующий таймлайн для контекста |

**Температура:** 0.5 | **Макс. токенов:** 1024 | **Приоритет:** 5

---

### Планер (`story-planner`)

**Описание:** Планирование сюжетных арок. Планирует квесты и развитие сюжета.

**Переменные шаблона:**
`{world_state}` `{characters}` `{events}` `{quests}`

| Переменная | Описание |
|-----------|----------|
| `{characters}` | Активные персонажи в мире |
| `{quests}` | Текущие активные квесты |

**Формат вывода:**
```json
{"arc": "описание", "quests": [{"title": "", "description": "", "objectives": [""]}], "hooks": [""]}
```

**Температура:** 0.7 | **Макс. токенов:** 2048 | **Приоритет:** 6

---

### Социальный симулятор (`social-sim`)

**Описание:** Социальная динамика. Моделирует отношения NPC и взаимодействия.

**Переменные шаблона:**
`{characters}` `{relationships}` `{context}`

| Переменная | Описание |
|-----------|----------|
| `{relationships}` | Текущий граф отношений между персонажами |
| `{context}` | Социальный контекст (встреча, конфликт, альянс и т.д.) |

**Температура:** 0.6 | **Макс. токенов:** 1024 | **Приоритет:** 4

---

### Менеджер злодеев (`villain`)

**Описание:** Управление антагонистами. Планирует действия злодеев и их schemes.

**Переменные шаблона:**
`{villain}` `{world_state}` `{recent_actions}`

| Переменная | Описание |
|-----------|----------|
| `{villain}` | Профиль злодея (личность, цели, способности) |
| `{recent_actions}` | Недавние действия злодея в мире |

**Температура:** 0.8 | **Макс. токенов:** 2048 | **Приоритет:** 6

---

### Исследователь (`researcher`)

**Описание:** Проверка фактов, валидация реализма и исследования для построения мира.

**Переменные шаблона:**
`{task}` `{world_context}`

| Переменная | Описание |
|-----------|----------|
| `{task}` | Задача исследования (проверка рецепта, валидация персонажа, обогащение сцены, проверка факта) |

**Формат вывода:**
```json
{"verdict": "plausible|questionable|unrealistic", "confidence": 0.0-1.0, "issues": [], "suggestions": [], "enrichedDetails": ""}
```

**Температура:** 0.3 | **Макс. токенов:** 2048 | **Приоритет:** 3 (наименьший)

---

## Руководство по температуре

| Значение | Эффект | Использовать для |
|----------|--------|------------------|
| 0.1 - 0.3 | Точный, детерминированный | Исследования, проверка фактов |
| 0.4 - 0.6 | Сбалансированный | Летописец, социальная симуляция |
| 0.7 - 0.8 | Креативный | Нарратив, диалоги NPC, схемы злодеев |

## Использование @agent в чате

Отправьте приватное сообщение любому агенту из чата:

```
@narrator опиши атмосферу древнего леса на закате
@director предложи драматический сюжетный поворот
@researcher исторически точен ли этот средневековый меч?
@chronicler резюмируй что произошло за последний час
```

Ответы отмечаются синей полоской слева и именем агента в скобках.

### Инъекция языковой инструкции

Ответы LLM автоматически соответствуют выбранному языку интерфейса. Языковая инструкция встраивается в промпты агентов при создании мира через `seedWorldAgents()`, а также добавляется во время выполнения через `getLanguageInstruction()`:

| Язык | Вставляемый текст |
|------|-------------------|
| en | `IMPORTANT: Always respond in English.` |
| ru | `ВАЖНО: Всегда отвечай на русском языке.` |
| de | `WICHTIG: Antworte immer auf Deutsch.` |
| fr | `IMPORTANT: Réponds toujours en français.` |
| es | `IMPORTANTE: Responde siempre en español.` |
| ja | `重要：常に日本語で回答してください。` |
| zh | `重要：请始终用中文回复。` |

При создании мира `seedWorldAgents()` записывает всех 14 агентов с языковой инструкцией, добавленной к системному промпту. Это гарантирует, что новые миры начинают с правильной языковой изоляции. Рантайм-функция `getLanguageInstruction()` используется в `dialogue-context.ts` для динамических диалогов NPC.

### API-эндпоинты для промптов

| Метод | Путь | Описание |
|-------|------|----------|
| `GET` | `/api/agents` | Список всех агентов (принимает `?world=`) |
| `GET` | `/api/agents/:id` | Получить конфигурацию агента (принимает `?world=`) |
| `PUT` | `/api/agents/:id` | Обновить конфигурацию агента (принимает `?world=`) |
| `PUT` | `/api/agents/:id/prompts` | Обновить промпты (принимает `?world=`) |
| `GET` | `/api/agents/:id/prompts/:lang` | Получить промпты для конкретного языка |
| `PUT` | `/api/agents/:id/prompts/:lang` | Создать или обновить промпты для конкретного языка |

**Параметры запроса:**
- `world` — необязательный, по умолчанию активный мир из настроек. Все эндпоинты агентов поддерживают `?world=` для операций с конкретным миром без переключения активного мира.

## Приоритет

Агенты с более высоким приоритетом обрабатываются первыми при очереди LLM-запросов.

| Агент | Приоритет |
|-------|-----------|
| narrator | 10 (наивысший) |
| npc | 9 |
| director | 8 |
| scene | 7 |
| story-planner | 6 |
| villain | 6 |
| chronicler | 5 |
| social-sim | 4 |
| researcher | 3 (наименьший) |

## Специализированные агенты (v0.26.0)

Следующие специализированные агенты подключены в `RoleplayEngine` и доступны через `engine.<agent>`:

| Агент | Поле | Назначение |
|-------|------|------------|
| **CartographerAgent** | `engine.cartographer` | Информация о локациях/географии — расстояния, пути, рельеф, достопримечательности |
| **HistorianAgent** | `engine.historian` | История мира, хронология, прошлые события, повествование о лоре |
| **LorekeeperAgent** | `engine.lorekeeper` | Факты мира, правила магической системы, информация о расах, установленный канон |
| **MerchantAgent** | `engine.merchant` | Торговля NPC-торговцев, цены, управление инвентарём |
| **QuestGiverAgent** | `engine.questGiver` | Генерация квестов на основе состояния мира, уровня игрока, сюжетных нитей |

Каждый специализированный агент принимает только `LLMQueue` в качестве зависимости и генерирует текст через собственные промпты.

## Система диалогов (v0.26.0)

Новая `DialogueManager` + `DialogueContext` для структурированных диалогов NPC:

| Возможность | Описание |
|-------------|----------|
| **Управление сессиями** | Жизненный цикл: Приветствие → Активный → Прощание |
| **Учитывает отношения** | Приветствия и доступность тем для друзей/нейтральных/врагов |
| **Феодальная иерархия** | Особые приветствия для лорда/вассалов |
| **Тематический выбор** | личное, фракция, квест, торговля, бой, ремесло, слухи, сплетни и т.д. |
| **Запись в память** | Резюме диалогов сохраняются в долгосрочную память NPC |

Доступ через `engine.dialogueManager` (требуется наличие `npcRuntime`).
