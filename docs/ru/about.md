# О TrueNeverStory — Обоснование дизайна

## Языковая стратегия: English внутри, перевод на границе

### Почему английский для обработки агентами

TrueNeverStory использует архитектуру **«English inside, translate at boundary»** по нескольким критическим причинам:

1. **Качество LLM** — Большие языковые модели работают лучше всего на английском, их основном языке обучения. Использование английского для внутренней обработки обеспечивает:
   - Более стабильное качество нарратива
   - Лучшее понимание сложных промптов
   - Меньше галлюцинаций и стилистических несоответствий
   - Доступ ко всему спектру литературных отсылок

2. **Экономия токенов** — английский обычно на 20-40% эффективнее по токенам, чем другие языки, для того же семантического содержания. Это означает:
   - Больше контекста помещается в окно промпта
   - Ниже стоимость API-запросов
   - Быстрее время обработки

3. **Литературное богатство** — Исходные материалы (Библия, классика Гутенберга) в основном на английском или имеют канонические английские переводы. Обработка на английском сохраняет:
   - Прямой доступ к архетипическим паттернам
   - Стилистическую аутентичность исходных текстов
   - Нюансированное эмоциональное и тематическое содержание

### Конвейер перевода

```
Ввод пользователя (любой язык)
    ↓
TranslationService.translateToEnglish()
    ↓
Разбор намерений (английский)
    ↓
Обработка агентами (английский)
    ↓
Генерация ответа (английский)
    ↓
TranslationService.translate()
    ↓
Вывод пользователю (оригинальный язык)
```

**Ключевые решения:**
- Перевод происходит **один раз на входе** и **один раз на выходе**
- Всё внутреннее состояние, память и обработка остаются на английском
- Агенты никогда не видят и не производят неанглийский текст напрямую
- Переводы UI отделены от переводов контента (i18n vs TranslationService)

## Архитектура литературной базы данных: экономия токенов через предобработку

### Проблема

Генерация богатых, литературных нарративов с нуля требует:
- Больших промптов с примерами стиля
- Множественных вызовов LLM для разных аспектов (сюжет, стиль, эмоции)
- Высокого потребления токенов для качественного результата

### Решение: офлайн-компиляция литературы

TrueNeverStory предобрабатывает литературные источники в структурированные SQLite базы данных **до деплоя**:

```
Исходные тексты → LiteraryCompiler → SQLite базы → Запросы в runtime
     ↓                ↓                  ↓              ↓
  Bible.db      4-проходный парсер   FTS5 индекс    Миллисекундные
  Gutenberg     (драматургический,   квест-шаблоны   запросы
  Классика      стилистический,      стилистические
                эмоциональный,       паттерны
                метаданные)
```

### Типы баз данных

| База данных | Источник | Содержание | Назначение |
|-------------|----------|------------|------------|
| `bible.db` | Библейские тексты | Квест-шаблоны, архетипы, моральные дилеммы | Нарративная структура |
| `gutenberg.db` | Project Gutenberg | Стилистические паттерны, сенсорные описания, темп | Литературное качество |
| `literary.db` | Компилированный вывод | Унифицированные шаблоны с FTS5 поиском | Доступ в runtime |

### Экономия токенов

**Без предобработки:**
```
Промпт: "Сгенерируй квест о предательстве в стиле древней эпической литературы..."
Токены: ~500-800 для промпта + ~300-500 для ответа = ~800-1300 токенов
```

**С предобработкой:**
```
Запрос: db.queryTemplates({ archetype: 'betrayal', mood: 'epic' })
Токены: ~50 для запроса + ~200-300 для ответа = ~250-350 токенов
```

**Экономия: снижение на 60-75% использования токенов на один нарративный элемент.**

## Богатые литературные источники

### Библейские архетипы

Библия предоставляет **проверенные временем нарративные структуры**, которые резонируют во всех культурах:

| Архетип | Источник | Паттерн | Современное применение |
|---------|----------|---------|----------------------|
| **Побег** | Исход 14 | Лидер → Тиран → Препятствие → Вмешательство → Свобода | Квесты мятежа, побег из тюрьмы |
| **Суд** | 3 Царств 3 | Спор → Мудрый правитель → Скрытая истина → Справедливость | Придворные интриги, моральные дилеммы |
| **Наследие** | Лука 15 | Блудный → Растратил → Возвращение → Принятие | Арки искупления, семейная драма |
| **Взлёт-Падение-Взлёт** | Бытие 37-50 | Избранник → Предан → Страдание → Возвышение → Примирение | Арки развития персонажа |
| **Стойкость** | Иов | Страдание → Сомнение → Настойчивость → Восстановление | Испытание решимости игрока |
| **Освобождение** | Судьи | Угнетение → Призыв → Сбор → Победа | Военные кампании, истории революции |

**Почему библейские паттерны работают:**
- **Узнаваемость** — Игроки интуитивно понимают эти структуры
- **Моральная сложность** — Библейские нарративы редко имеют простые разделения добро/зло
- **Эмоциональная глубина** — Темы потери, надежды, предательства, искупления
- **Масштабируемая драма** — Работает для интимных историй и эпических кампаний

### Стилистические паттерны Гутенберга

Project Gutenberg предоставляет **века литературного мастерства**:

| Эпоха | Авторы | Стилистические элементы | Применение |
|-------|--------|------------------------|------------|
| **Готика** | По, Шелли, Стокер | Тёмная атмосфера, сенсорный страх, психологическое напряжение | Ужасы, детективы |
| **Викторианская** | Диккенс, сёстры Бронте | Социальный комментарий, детальные описания, моральная сложность | Социальные интриги |
| **Эпическая** | Гомер, Мильтон | Грандиозный масштаб, героический язык, мифический резонанс | Войны, квесты |
| **Романтическая** | Байрон, Китс | Эмоциональная интенсивность, образы природы, страсть | Истории любви, личная драма |

**Процесс дексификации:**
1. Извлечение структурных паттернов (длина предложений, ритм, словарь)
2. Удаление имён персонажей и конкретных отсылок
3. Сохранение сенсорных маркеров и эмоционального тона
4. Создание многоразовых шаблонов с переменными

## Индивидуальность и характер NPC

### Многоуровневая система персонажей

Каждый NPC имеет **четыре уровня глубины**:

```
L1: Базовая информация (имя, роль, местоположение)
    ↓
L2: Личность (черты, причуды, речевые паттерны)
    ↓
L3: Скрытые мотивации (тайные цели, страхи, желания)
    ↓
L4: Динамическое состояние (отношения, память, эмоциональное состояние)
```

### Источники персонажей

| Источник | Вклад | Пример |
|----------|-------|--------|
| **Библейские архетипы** | Моральные рамки, паттерны лояльности | Верность Руфи, амбиции Давида |
| **Персонажи Гутенберга** | Речевые паттерны, социальное поведение | Социальные карьеристы Диккенса, страстные души сестёр Бронте |
| **Исторические паттерны** | Политическое поведение, динамика фракций | Придворные интриги, гильдейская политика |
| **Психологические модели** | Согласованность личности, эмоциональные реакции | Черты Большой Пятёрки, стили привязанности |

### Система памяти NPC

```
Краткосрочная: Последние 3 взаимодействия (немедленный контекст)
    ↓
Среднесрочная: Значимые события (изменения отношений)
    ↓
Долгосрочная: Основные воспоминания (формирующие опыт)
    ↓
Семантическая: Запросы на основе эмбеддингов (контекстуальная память)
```

**Влияние памяти:**
- Тон и словарь диалога
- Уровень доверия и готовность помочь
- Стиль приветствия (тёплый, холодный, испуганный)
- Доступность тем (личные, фракционные, квестовые)

### Экономическое и социальное поведение

NPC имеют **реалистичное экономическое поведение** на основе исторических паттернов:

| Поведение | Источник | Реализация |
|-----------|----------|------------|
| **Торговля** | Средневековые купеческие гильдии | Спрос/предложение, ценообразование на основе репутации |
| **Ремесло** | Исторические мастерские ремесленников | Уровни навыков, качество материалов, затраты времени |
| **Социальная динамика** | Феодальная иерархия | Отношения лорд/вассал, лояльность фракции |
| **Политические интриги** | Придворная политика | Тайные союзы, торговля информацией |

## Механика сюжетных поворотов

### Выбор нарративного паттерна

Когда игрок совершает действие, система:

1. **Анализирует намерение** — Что игрок пытается сделать?
2. **Моделирует исход** — Что реалистично произойдёт?
3. **Выбирает архетип** — Какой библейский/литературный паттерн подходит?
4. **Применяет стиль** — Какая литературная эпоха/настроение соответствует?
5. **Генерирует прозу** — Объединяет паттерн + стиль + контекст

### Пример: Предательство союзника

```
Намерение: предательство
Моделирование: Отношения разрушены, напряжение фракций растёт
Архетип: Бытие 37 (Иосиф, проданный братьями)
Стиль: Викторианская социальная драма
Результат: Нарратив, исследующий последствия предательства с диккенсовской социальной детализацией
```

### Динамическая сложность

Квест-шаблоны включают **оценки моральной неоднозначности** (0-1):
- 0.0 — Чёткое добро/зло (детские истории)
- 0.5 — Сложный выбор (стандартная RPG)
- 1.0 — Нет правильного ответа (зрелые нарративы)

## Архитектура производительности

### Почему предобработка побеждает

| Подход | Задержка | Стоимость токенов | Качество |
|--------|----------|-------------------|----------|
| **LLM на лету** | 2-5 секунд | Высокое | Переменное |
| **Предобработанная БД** | <100мс | Минимальное | Стабильное |
| **Гибрид (TNS)** | <100мс + полировка | Низкое | Высокое |

### Гибридный подход

1. **Запрос к БД** — Получение структурированного шаблона (миллисекунда)
2. **Подстановка переменных** — Заполнение контекста (микросекунда)
3. **Применение стиля** — Применение литературных паттернов (миллисекунда)
4. **Полировка LLM** — Финальная генерация прозы (опционально, для критических моментов)

Это даёт нам **скорость базы данных** с **качеством LLM** там, где это важно.

## Правила мира и экономика

### Феодальная иерархия

TrueNeverStory реализует **10-уровневую феодальную систему рангов**, которая управляет богатством, налогами, привилегиями и социальными взаимодействиями:

| Ранг | Мин. богатство | Стража | Базовый налог | Жалованье | Может давать взятки | Может брать взятки |
|------|----------------|--------|---------------|-----------|---------------------|-------------------|
| **Раб** | 0 | 0 | 100% | 0 | Нет | Нет |
| **Крестьянин** | 0 | 0 | 90% | 0 | Да | Нет |
| **Баронет** | 100,000 | 50 | 30% | 0 | Да | Да |
| **Барон** | 500,000 | 200 | 28% | 0 | Да | Да |
| **Виконт** | 2,000,000 | 1,000 | 25% | 0 | Да | Да |
| **Граф** | 10,000,000 | 5,000 | 22% | 0 | Да | Да |
| **Маркиз** | 50,000,000 | 20,000 | 20% | 0 | Да | Да |
| **Герцог** | 200,000,000 | 100,000 | 18% | 0 | Да | Да |
| **Король** | 1,000,000,000 | 500,000 | 15% | 0 | Да | Да |
| **Император** | 10,000,000,000 | 2,000,000 | 10% | 0 | Да | Да |

**Ключевые правила:**
- **Рабы** не могут участвовать в экономике взяток (нет свободы воли)
- **Крестьяне** могут давать взятки, но не могут получать (нет власти)
- **Высшие ранги** платят меньше налогов (скидка за власть), но имеют более высокие расходы на содержание
- **Пороги богатства** должны быть достигнуты для повышения в ранге

### Система налогов

Система налогов **динамическая** и зависит от множества факторов:

```
Эффективный налог = Базовый налог × (1 - Скидка за власть - Скидка за популярность)
```

**Компоненты:**
- **Базовый налог** — Определяется рангом (90% для крестьян, 10% для императоров)
- **Скидка за власть** — До 90% снижения на основе политической власти (власть / 10,000)
- **Скидка за популярность** — До 30% снижения на основе общественного одобрения (популярность / 3,000)

**Поток налогов:**
```
Доход NPC → Расчёт налога → Сбор в казну → Казна лорда
     ↓              ↓              ↓              ↓
  Ранго-       Динамические    Обработка за   Накопление
  зависимое    ставки          ход с проверкой по феодальной
  жалованье                    лояльности     цепочке
```

**Последствия высоких налогов:**
- **Риск предательства** = (Налоговая нагрузка + Взятки) / Доход × (1 - Лояльность / 1000)
- Высокая налоговая нагрузка увеличивает вероятность мятежа
- Лояльность действует как буфер против предательства

### Экономика взяток

Взятки — это **механика первого класса**, которая влияет на политику, лояльность и динамику власти:

**Типы взяток:**
| Тип | Назначение | Уровень риска |
|-----|------------|---------------|
| **Защита** | Избежать наказания или преследования | Средний |
| **Услуга** | Получить привилегированное отношение | Низкий |
| **Молчание** | Скрыть секреты | Высокий |
| **Доступ** | Получить доступ к ограничённым зонам или людям | Средний |
| **Продвижение** | Повысить ранг или должность | Высокий |
| **Освобождение** | Избежать налогов или обязательств | Очень высокий |

**Формула риска взятки:**
```
Риск = Базовый риск (10%) + Риск суммы (сумма / 10,000) + Риск свидетелей (свидетели × 15%) - Навык получателя (интриги × 0.1%)
```

**Факторы риска:**
- **Сумма** — Большие взятки рискованнее
- **Свидетели** — Больше наблюдателей увеличивают шанс обнаружения
- **Интриги получателя** — Умелые чиновники могут скрывать коррупцию
- **Тип взятки** — Некоторые типы inherently рискованнее

**Экономическое влияние:**
- Взятки **снижают лояльность** (коррупция разъедает доверие)
- Взятки **увеличивают богатство** получателей
- Взятки **увеличивают власть** (1% суммы взятки конвертируется в власть)
- Взятки **снижают популярность** (общество не одобряет коррупцию)

### Экономические циклы (модель Иосифа)

На основе **библейской модели Иосифа** (Бытие 41) экономика циклически проходит три фазы:

**Цикл фаз:**
```
Изобилие (30 дней) → Переход (10 дней) → Голод (20 дней) → Изобилие...
```

**Эффекты фаз:**
| Фаза | Модификатор цен | Изменение резерва | Нарративные события |
|------|-----------------|-------------------|---------------------|
| **Изобилие** | 0.8× (дёшево) | +20% за ход | Урожай, торговые бумы |
| **Переход** | 1.0× (норма) | Стабильно | Рыночная неопределённость |
| **Голод** | 2.0× (дорого) | -30% за ход | Засуха, чума, война |

**Стратегические последствия:**
- **Накапливайте резервы** во время изобилия для выживания в голод
- **Планирование цен** требует предвидения переходов фаз
- **Решения игрока** могут влиять на длительность и тяжесть фаз

### Юбилейная система

Каждые **50 лет** происходит **юбилейное событие**, вызывающее масштабный экономический сброс:

**Эффекты юбилея:**
1. **Сброс долгов** — Все непогашенные долги прощаются
2. **Возврат земель** — Все заложенные земли возвращаются владельцам
3. **Буст лояльности** — +30% бонус лояльности для всех NPC (длится 10 дней)

**Историческая основа:**
На основе **библейского юбилея** (Левит 25), который предотвращал permanentную бедность и поддерживал социальную мобильность.

**Стратегические последствия:**
- **Тайминг долгов** — Избегайте lending вблизи юбилейных лет
- **Приобретение земель** — Временное владение создаёт интересные динамики
- **Социальная стабильность** — Юбилей предотвращает extreme концентрацию богатства

### Дилеммы фракционных налогов

Когда две фракции имеют конфликтующие налоговые претензии, система генерирует **моральные дилеммы** для игрока:

**Генерация дилемм:**
- 30% шанс за экономический ход
- Минимальный кулдаун 30 дней между дилеммами
- Суммы налогов от 100 до 1,000 золотых

**Выбор игрока:**
| Выбор | Лояльность фракции A | Лояльность фракции B | Репутация |
|-------|---------------------|---------------------|-----------|
| **Заплатить фракции A** | +50 | -30 | Нейтрально |
| **Заплатить фракции B** | -30 | +50 | Нейтрально |
| **Отказать обоим** | -20 | -20 | -10 (ненадёжный) |

**Нарративная интеграция:**
Каждая дилемма генерирует контекстную историю, объясняющую конфликт, заставляя игрока делать значимые политические выборы.

### Правила труда фракций

Каждая фракция может устанавливать **индивидуальную трудовую политику**, влияющую на зарплаты и лояльность:

**Параметры правил труда:**
- **Фиксированные зарплаты** — Да/Нет (предсказуемо vs. на основе производительности)
- **Размер зарплаты** — Базовая оплата за рабочий период
- **Модификатор лояльности** — Бонус/штраф к лояльности работника

**Расчёт зарплаты:**
```
Итоговая зарплата = Базовая зарплата × Отработанные часы × Производительность × Модификатор фракции
```

**Конфликты лояльности:**
Когда NPC работают на несколько фракций, могут возникать конфликты лояльности:
- **Двойная лояльность** снижает эффективность
- **Смена фракции** имеет репутационные издержки
- **Остановки работы** происходят во время конфликтов

### Работорговля

**Морально сложная** система, отражающая исторические динамики власти:

**Свойства рабов:**
- **Здоровье** — Физическое состояние (влияет на ценность)
- **Опыт** — Навыки и знания (увеличивает ценность)
- **Пороки** — Жадность, гнев, лень (снижает ценность)
- **Семья** — Рабы могут иметь семьи (создаёт обязательства)

**Формула ценности раба:**
```
Ценность = 100 + (Опыт × 0.1) + (Здоровье × 0.05) - Штрафы за пороки
```

**Жизненный цикл раба:**
1. **Рабство** — Через долги, захват или рождение
2. **Труд** — Производит товары (300-1000 единиц за ход)
3. **Мятеж** — Возможен, если стража слаба (соотношение сил определяет успех)
4. **Освобождение** — Стоит 200 золотых, даёт статус крестьянина

**Этические соображения:**
- Система создана быть **морально некомфортной**
- Выбор игрока имеет **реальные последствия** для NPC-рабов
- **Механика мятежа** гарантирует, что рабство никогда не бывает "безопасным"
- **Пути освобождения** предоставляют возможности для морального искупления

### Интеграция с социальным графом

Экономика интегрируется с **социальным графом** для реалистичных динамик власти:

**Феодальные отношения:**
```
Вассал → Лорд (налог + военное обязательство)
  ↓
Цепочка командования (несколько уровней)
  ↓
Накопление потока налогов (общий сбор налогов)
```

**Экономика фракций:**
- **Экономические фракции** контролируют торговлю и ресурсы
- **Военные фракции** обеспечивают безопасность (за плату)
- **Религиозные фракции** влияют на моральные выборы
- **Криминальные фракции** действуют вне легальной экономики

**Система репутации:**
- **Общественное одобрение** влияет на ставки налогов
- **Положение во фракции** определяет доступ к ресурсам
- **Личная репутация** влияет на成功率 взяток

### Экономическая база данных

Все экономические данные хранятся в выделенной **SQLite базе данных** (`economic.db`):

**Таблицы:**
- `economic_cycles` — Отслеживание фаз с модификаторами цен
- `jubilee_events` — Исторические записи юбилеев
- `faction_labor_rules` — Трудовая политика по фракциям
- `faction_dilemmas` — История налоговых споров

**Производительность:**
- **Индексированные запросы** для быстрых экономических расчётов
- **Пакетная обработка** для NPC операций (старение, пороки, налоги, лояльность)
- **Ядра Mojo** для вычислительно-intensive операций (5× ускорение)

## Резюме

Архитектура TrueNeverStory сочетает:

1. **Обработку на английском** для качества LLM и экономии токенов
2. **Предобработанные литературные базы данных** для мгновенного доступа к богатым нарративным паттернам
3. **Многоуровневые системы NPC** для правдоподобных, запоминающихся персонажей
4. **Гибридную генерацию**, балансирующую скорость и качество

Результат — система, способная генерировать **нарративы литературного качества** на **скорости баз данных**, сохраняя **глубокую согласованность персонажей** и **значимый выбор игрока**.
