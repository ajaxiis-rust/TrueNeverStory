# Требования к железу и рекомендации по моделям

TrueNeverStory гибкая система — можно запускать на чём угодно, от Raspberry Pi до мульти-GPU сервера. Выбирайте конфигурацию под доступное железо.

---

## Минимальные требования

| Компонент | Минимум | Рекомендуется |
|-----------|---------|---------------|
| **RAM** | 4 GB | 8+ GB |
| **CPU** | 2 ядра | 4+ ядра |
| **Диск** | 2 GB | 10+ GB |
| **GPU** | Не требуется | Любая GPU с VRAM |

---

## Профили конфигурации

### Профиль 1: Ультра-лёгкий (2-4 GB RAM)

**Для чего:** Старый ноутбук, VPS, Raspberry Pi 4+

```
Intent Parser:    Gemma 3 1B (Q4_K_M) — 760 MB
Переводы:         NLLB-200 600M (Q4_K_M) — 340 MB
Нарратив:         Облачный API (Gemini Flash)
Embeddings:       Hash-fallback (модель не нужна)
```

**Общая RAM:** ~1.1 GB  
**Скорость:** Медленно (3-5 токенов/сек на CPU)  
**Качество:** Приемлемое  
**Приватность:** Частичная (нарратив через облако)

---

### Профиль 2: Сбалансированный (4-8 GB RAM)

**Для чего:** Современный ноутбук, десктоп, небольшой сервер

```
Всё-в-одном:      Llama 3.2 3B (Q4_K_M) — 2 GB
                  ИЛИ Qwen 2.5 3B (Q4_K_M) — 2 GB
Переводы:         NLLB-200 600M (Q4_K_M) — 340 MB
Embeddings:       BGE M3 (Q4_K_M) — 438 MB
```

**Общая RAM:** ~2.8 GB  
**Скорость:** Умеренно (5-10 токенов/сек на CPU)  
**Качество:** Хорошее  
**Приватность:** Полная (всё локально)

---

### Профиль 3: Качество (8-16 GB RAM)

**Для чего:** Игровой ПК, рабочая станция, выделенный сервер

```
Всё-в-одном:      Gemma 3 4B (Q4_K_M) — 2.3 GB
                  ИЛИ Qwen 2.5 7B (Q4_K_M) — 4.4 GB
Переводы:         NLLB-200 600M (Q8_0) — 620 MB
Embeddings:       BGE M3 (Q8_0) — 635 MB
```

**Общая RAM:** ~3.6-5.5 GB  
**Скорость:** Хорошая (10-20 токенов/сек на CPU, 30+ на GPU)  
**Качество:** Высокое  
**Приватность:** Полная

---

### Профиль 4: GPU-ускорение (4+ GB VRAM)

**Для чего:** Игровой ПК с GPU, рабочая станция с дискретной GPU

```
LLM:              Llama 3.1 8B (Q4_K_M) — 5 GB VRAM
Переводы:         Та же модель (многоязычная)
Embeddings:       BGE M3 (Q8_0) — CPU offload
```

**Общая VRAM:** ~5 GB  
**Скорость:** Быстро (30-50 токенов/сек)  
**Качество:** Отличное  
**Приватность:** Полная

## Примеры конфигураций

### Пример 1: Бюджетный VPS (4 GB RAM)

```bash
# .env
WORLD_LLM_BASE_URL=http://localhost:11434/v1
WORLD_LLM_MODEL=llama3.2:3b

# Настройки Ollama
OLLAMA_NUM_PARALLEL=1
OLLAMA_MAX_LOADED_MODELS=2
```

**Установка моделей:**
```bash
ollama pull llama3.2:3b
ollama pull nllb-200-distilled-600m
```

---

### Пример 2: Десктоп с GPU (8 GB VRAM)

```bash
# .env
WORLD_LLM_BASE_URL=http://localhost:11434/v1
WORLD_LLM_MODEL=llama3.1:8b
WORLD_EMBEDDING_MODEL=bge-m3
WORLD_EMBEDDING_BASE_URL=http://localhost:5002
```

---

### Пример 3: Облачный гибрид

```bash
# .env — Нарратив через облако
WORLD_LLM_BASE_URL=https://generativelanguage.googleapis.com/v1beta
WORLD_LLM_MODEL=gemini-2.0-flash
WORLD_LLM_API_KEY=your-key

# Локально для переводов (приватность)
OLLAMA_NUM_PARALLEL=1
```

**В настройках агентов:**
- Narrator → Google Gemini (облако)
- Translation → Ollama NLLB-200 (локально)
- Intent Parser → Ollama Llama 3.2 3B (локально)

## Советы по оптимизации

1. **Только CPU:** Используйте квантизацию Q4_K_M для лучшего соотношения скорость/качество
2. **GPU:** Q5_K_M или Q8_0 для лучшего качества (если позволяет VRAM)
3. **Параллельные запросы:** Установите `WORLD_LLM_MAX_CONCURRENT=1` на CPU
4. **Embeddings:** Используйте hash-fallback если не хватает RAM (отключите модель embeddings)
5. **Переводы:** NLLB-200 оптимизирован для переводов, а не для общего чата
